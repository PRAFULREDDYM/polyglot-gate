# API Reference

The examples in this file were generated by running the backend locally with `DATABASE_URL=sqlite:////private/tmp/polyglot_docs_api.db`, `LLM_PROVIDER=mock`, and `uvicorn app.main:app --host 127.0.0.1 --port 8000`, then calling each endpoint with `curl`. Timestamps and IDs are therefore real examples from that local run.

## Health

**Method:** `GET`

**Path:** `/health`

**Purpose:** Confirms the API process is reachable.

**Request body:** none.

**Response body:**

```json
{"status":"ok","service":"polyglot-gate"}
```

**Meaningful non-200 statuses:** none from application logic. If the server is not running, clients will see a connection error rather than an API response.

## Evaluate Prompt

**Method:** `POST`

**Path:** `/api/evaluate`

**Purpose:** Generates one model output, scores it, stores `Prompt`/`Run`/`Score` rows, records metrics, and returns the score breakdown.

**Request body:**

```json
{
  "prompt_text": "Write a concise greeting for a new user.",
  "locale": "en",
  "reference_answer": "Welcome to your new account.",
  "model_name": null
}
```

**Response body:**

```json
{"run_id":1,"prompt_id":1,"output_text":"[MOCK-en] Write a concise greeting for a new user.","locale":"en","latency_ms":10,"scores":{"correctness":0.083,"formatting":1.0,"locale_coverage":1.0,"regression_pass":true,"overall":0.633},"passed":false,"failure_reasons":["correctness below threshold (0.08)"]}
```

**Meaningful non-200 statuses:**

- `422 Unprocessable Entity`: request validation fails, for example `prompt_text` is empty or whitespace-only.
- `500 Internal Server Error`: `LLM_PROVIDER=anthropic` is selected but `ANTHROPIC_API_KEY` is missing, or the provider call raises an unexpected SDK/runtime error.

## Evaluate Multi-Locale

**Method:** `POST`

**Path:** `/api/evaluate/multi-locale`

**Purpose:** Runs the same prompt through multiple locales, stores all runs under one prompt ID, and reports cross-locale consistency.

**Request body:**

```json
{
  "prompt_text": "Summarize the checkout error in one sentence.",
  "locales": ["en", "es", "fr"],
  "reference_answer": "The checkout failed because payment could not be completed."
}
```

**Response body:**

```json
{"prompt_id":2,"results":[{"locale":"en","run_id":2,"scores":{"correctness":0.133,"formatting":1.0,"locale_coverage":1.0,"regression_pass":true,"overall":0.653},"passed":false},{"locale":"es","run_id":3,"scores":{"correctness":0.133,"formatting":1.0,"locale_coverage":0.0,"regression_pass":true,"overall":0.453},"passed":false},{"locale":"fr","run_id":4,"scores":{"correctness":0.133,"formatting":1.0,"locale_coverage":1.0,"regression_pass":true,"overall":0.653},"passed":false}],"consistency_score":0.8,"worst_locale":"es","summary":"es scored 0.20 lower than en"}
```

**Meaningful non-200 statuses:**

- `422 Unprocessable Entity`: `prompt_text` is blank or `locales` is empty or contains blank values.
- `500 Internal Server Error`: the configured LLM provider cannot generate a response.

## Regression Suite

**Method:** `GET`

**Path:** `/api/evaluate/regression-suite`

**Purpose:** Runs every fixture in `backend/tests/fixtures/regression_cases.json` through the evaluation service and reports failures below each case threshold.

**Request body:** none.

**Response body:**

```json
{"total":6,"passed":6,"failed":0,"failures":[]}
```

**Meaningful non-200 statuses:**

- `500 Internal Server Error`: the configured LLM provider or database session fails while running the batch. In CI, this endpoint is smoke-tested with `LLM_PROVIDER=mock`.

## Evaluation History

**Method:** `GET`

**Path:** `/api/evaluate/history?limit=3&offset=0`

**Purpose:** Returns recent evaluation runs joined to their latest score, ordered newest first.

**Request body:** none.

**Response body:**

```json
[{"run_id":4,"prompt_id":2,"prompt_text":"Summarize the checkout error in one sentence.","locale":"en","output_text":"[MOCK-fr] Summarize the checkout error in one sentence.","overall":0.653,"passed":false,"created_at":"2026-06-26T21:15:16"},{"run_id":3,"prompt_id":2,"prompt_text":"Summarize the checkout error in one sentence.","locale":"en","output_text":"[MOCK-es] Summarize the checkout error in one sentence.","overall":0.453,"passed":false,"created_at":"2026-06-26T21:15:16"},{"run_id":2,"prompt_id":2,"prompt_text":"Summarize the checkout error in one sentence.","locale":"en","output_text":"[MOCK-en] Summarize the checkout error in one sentence.","overall":0.653,"passed":false,"created_at":"2026-06-26T21:15:16"}]
```

The optional `locale` query parameter filters on `Prompt.locale`. Multi-locale evaluations currently store all runs under one shared prompt row, so history reflects that prompt-level locale.

**Meaningful non-200 statuses:**

- `422 Unprocessable Entity`: `limit` is outside `1..200` or `offset` is negative.

## Moderate

**Method:** `POST`

**Path:** `/api/moderate`

**Purpose:** Applies YAML-backed moderation regex rules, writes an audit log row, and creates a policy `Issue` row when the action is `flag` or `block`.

**Request body:**

```json
{
  "text": "Please contact reviewer@example.com for escalation.",
  "locale": "en",
  "context": "support reply"
}
```

**Response body:**

```json
{"audit_id":1,"is_safe":false,"labels":["PII_EMAIL"],"policy_violations":[{"rule_id":"PII_EMAIL","rule_name":"Email address","severity":"medium"}],"action":"flag"}
```

**Meaningful non-200 statuses:**

- `422 Unprocessable Entity`: required fields are missing or request JSON is malformed.
- `500 Internal Server Error`: the YAML rule file cannot be loaded or a regex is invalid.

## Triage

**Method:** `POST`

**Path:** `/api/triage`

**Purpose:** Classifies a globalization issue, writes a routed `Issue` row, and returns the queue and suggested fix.

**Request body:**

```json
{
  "run_id": null,
  "prompt_text": "Write a checkout error in Spanish.",
  "output_text": "Checkout failed",
  "locale": "es",
  "reported_issue": "untranslated fallback text"
}
```

**Response body:**

```json
{"issue_id":2,"issue_category":"translation","confidence":0.8,"suggested_fix":"Re-run generation with an explicit instruction to respond only in es and verify no English fallback strings remain.","route_to":"i18n-team"}
```

**Meaningful non-200 statuses:**

- `422 Unprocessable Entity`: required fields such as `prompt_text` or `output_text` are missing, or `run_id` is not an integer/null.
- `500 Internal Server Error`: the issue insert fails, for example if a non-null `run_id` references no existing run and SQLite foreign-key enforcement is enabled.

## Triage Issues

**Method:** `GET`

**Path:** `/api/triage/issues?limit=3&offset=0`

**Purpose:** Lists recent `Issue` rows in dashboard summary form, ordered newest first.

**Request body:** none.

**Response body:**

```json
[{"issue_id":2,"issue_category":"translation","confidence":0.8,"suggested_fix":"Re-run generation with an explicit instruction to respond only in es and verify no English fallback strings remain.","route_to":"i18n-team"},{"issue_id":1,"issue_category":"policy","confidence":0.0,"suggested_fix":"","route_to":"general-triage"}]
```

**Meaningful non-200 statuses:**

- `422 Unprocessable Entity`: `limit` is outside `1..200` or `offset` is negative.

## Metrics

**Method:** `GET`

**Path:** `/api/metrics`

**Purpose:** Returns in-process service counters and failure reason totals recorded since process start.

**Request body:** none.

**Response body:**

```json
{"services":{"evaluate":{"count":4,"avg_latency_ms":10.0,"pass_count":0,"fail_count":4},"moderate":{"count":1,"avg_latency_ms":10.0,"pass_count":0,"fail_count":1},"triage":{"count":1,"avg_latency_ms":4.0,"pass_count":1,"fail_count":0}},"failures_by_reason":{"correctness below threshold (0.08)":1,"correctness below threshold (0.13)":3,"PII_EMAIL":1}}
```

**Meaningful non-200 statuses:** none from application logic. Metrics are memory-local and reset when the process restarts.
